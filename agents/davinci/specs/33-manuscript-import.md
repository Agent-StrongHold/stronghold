# 33 — Manuscript Import

**Status**: P0 / Content phase. Required for early readers + illustrated
novels.
**One-liner**: ingest a Word/PDF/Markdown/Google Doc manuscript, auto-paginate
into a Document, mark scene breaks for spot illustration, set typography per
age band.

## Problem it solves

For early-reader books and illustrated novels, the manuscript exists outside
Da Vinci — in Word, Google Docs, Scrivener. The user has 5,000–30,000 words
of polished text. Without import, they'd manually retype or copy-paste page
by page. With import, the entire manuscript becomes a paginated Document
ready for illustration in one step.

## Data flow

```
def import_manuscript(file, doc_kind, age_band, [layout_kind]) -> Document:
    text_blocks = parse_to_blocks(file)              # paragraph stream w/ structure
    structure = detect_structure(text_blocks)        # chapters, sections, headings
    pages = paginate(text_blocks, age_band, layout_kind)
    illustrations = mark_scene_breaks(text_blocks)
    document = build_document(doc_kind, pages, structure, illustrations)
    apply_typography_defaults(document, age_band)
    return document
```

## Block model

```
TextBlock (frozen):
  kind: BlockKind                 # PARAGRAPH | HEADING | LIST | QUOTE | DIALOGUE | SCENE_BREAK
  level: int = 0                  # heading level
  content: str
  source_offset: tuple[int, int]  # (start, end) in original file
  metadata: Mapping[str, Any]

BlockKind (StrEnum):
  PARAGRAPH | HEADING | LIST_ITEM | LIST | QUOTE | DIALOGUE
  | SCENE_BREAK | CHAPTER_BREAK | PAGE_BREAK
  | FRONT_MATTER | BACK_MATTER
```

## Pagination algorithm

Per `AgeBand`, words-per-page targets and layout assignments differ:

| AgeBand | Words/page | Layout |
|---|---|---|
| 5_7 | 30-60 | ART_WITH_BODY (large illustration above small body) |
| 7_9 | 60-120 | ART_WITH_BODY (smaller illustration) |
| 9_12 | 200-300 | VIGNETTE (spot illustration + body wrap) |
| TEEN | 250-400 | TEXT_ONLY with chapter-start ART_WITH_BODY |

Algorithm:
1. Group blocks until target word count reached
2. Prefer breaks at PARAGRAPH boundaries; fallback at sentence
3. Never break mid-sentence
4. CHAPTER_BREAK forces a new page (TEXT_ONLY chapter-start layout)
5. SCENE_BREAK records an illustration marker, doesn't necessarily break

## Structure detection

Heuristics (heading→chapter, etc.):
- DOCX: use Word styles ("Heading 1" → chapter, "Heading 2" → section)
- Markdown: use # / ## levels
- Plain text: detect ALL CAPS lines (chapter), centred lines, "Chapter N",
  scene-break markers (`* * *`, `---`, `~`)

If detection is uncertain, surface a "review structure" UI step before
auto-paginating.

## Scene-break markers → illustration slots

Common scene-break markers add an SCENE_BREAK block, which produces an
illustration slot on the page where the break appears. Da Vinci proposes
illustration prompts based on:
- Surrounding text (3-5 sentences around the break)
- Vision-LLM extracts a scene description
- User reviews + edits before committing to generation (cost gate)

## Front/back matter handling

DOCX/EPUB front matter (title page, copyright, dedication) maps to layouts:

| Source content | Target layout |
|---|---|
| Title + subtitle + author | TITLE_PAGE |
| "Copyright …" | COPYRIGHT_PAGE |
| "For my..." | DEDICATION_PAGE |
| Table of contents | FRONT_MATTER (auto-generated by Da Vinci, not imported) |
| Acknowledgements | BACK_MATTER |
| Author bio | BACK_MATTER |

## Typography defaults

Applied at import per `AgeBand` (§25 accessibility defaults):

| Field | 5_7 | 7_9 | 9_12 |
|---|---|---|---|
| body font | Atkinson Hyperlegible | Atkinson Hyperlegible | Inter |
| body size | 18pt | 14pt | 12pt |
| line height | 1.6× | 1.5× | 1.4× |
| paragraph spacing | 12pt | 8pt | 6pt |
| chapter heading size | 36pt | 30pt | 24pt |
| dyslexia mode | recommended | recommended | optional |

User can override.

## Illustration placeholder strategy

Pages with SCENE_BREAK markers get a placeholder illustration layer with:
- Empty raster, marked PLACEHOLDER (slot_id = "illustration")
- `metadata.proposed_prompt` from Da Vinci's scene extraction
- Cost gate triggers when user clicks "generate"

This avoids a $5+ surprise at import time for a 200-illustration book.

## Edge cases

1. **Manuscript exceeds page limit for kind** — picture_book max 48; if
   import would exceed, warn user to use early_reader instead.
2. **No structure detected** — best-effort flat pagination; warn user that
   chapters/headings may not align.
3. **Source has unusual fonts** — fonts NOT imported; user picks from kit.
4. **Source has embedded images** — captured as separate uploads under
   asset library (§18); user decides where to place.
5. **Markdown with code blocks** — render as monospace text layer;
   children's books rarely have code; warn if found.
6. **DOCX with track-changes** — accept all changes; do not import comments.
7. **EPUB with XHTML CSS** — strip styles; reconstruct typography from age
   band defaults.
8. **PDF-with-text where text extraction is wonky** — surface to user before
   commit; suggest OCR (P2) or reformatted source.
9. **Scanned PDF (image-based)** — reject in P0; suggest OCR step (P2).
10. **Encoding errors / mojibake** — auto-detect via `chardet`; warn if
    confidence low.
11. **Scene-break heuristic produces too many illustrations** (every page) —
    cap at 1 per page; rest become text-only.
12. **Manuscript in a language without good Whisper support** (impacts
    audio §27) — flag at import; audio reach goal accounts for this.

## Errors

- `ManuscriptFormatUnsupportedError(ConfigError)`
- `ManuscriptParseError(ToolError)` — parse failed at offset
- `ManuscriptStructureUnclearError(ConfigError)` — heuristic confidence low
- `ManuscriptEncodingError(ToolError)`

## Test surface

- Integration: import a 5000-word fixture → ~80 pages at age 5_7;
